Cloning Objects

  • Spread Operator
  • Object.assign()
const food = {
        beef: 1,
        turkey: 2,
    }
    
    //Clone using the spread operator
    const cloneFood = {...food};
    
    //Clone using the Object.assign() method
    const cloneByObject = Object.assign({}, food);

These two operations will return us back "shallow clone". So essentially if we were to have nested objects in each other, it will only retain the first value. See the example below

const nestedObj = {
        flag: 'black',
        countryCulture: {
            food: 'nooodles',
            clothes: 'cool'
        }
    }
    const country = {...nestedObj}
    
    console.log(country.flag); // black
    console.log(country.countryCulture.food) // nooodles
    
    
    //Changing the values in the object and also in the nested object
    country.flag = 'blue';
    country.countryCulture.food = 'soup';
    
    console.log(country.countryCulture);
    console.log(nestedObj.countryCulture); // <- Our nestedObj has also been changed

When we use a shallow copy on the first level is copied whilst the deeper elements are referenced.

Deep Copy

A quick and easy way of doing a deep copy is using JSON.stringify() which converts the object to a string and then JSON.parse() converting the string back to an object.

const obj = {
        a: {
            b: {
                c: 2
            }
        }
    }
    
    // const objClone = {...obj};
    const objClone = JSON.parse(JSON.stringify(obj));
    objClone.c = 1;
    console.log(objClone);
    console.log(obj);

Otherwise, we will need to call it recursively so we are able to grab all the values that are sitting inside the nested loops.

Sorting Arrays

The array.sort() method sorts the array in-place and it does it by converting the elements to a string and then comparing their sequences. See the MDN example below

const months = ['March', 'Jan', 'Feb', 'Dec'];
    months.sort();
    console.log(months);
    // expected output: Array ["Dec", "Feb", "Jan", "March"]
    
    const array1 = [1, 30, 4, 21, 100000];
    array1.sort();
    console.log(array1);
    // expected output: Array [1, 100000, 21, 30, 4]

Notice how when we are trying to sort the numbers, it is not returning it to us numerical order, rather it is comparing each string a time and ordering it in that manner. Good explanation from MDN:

...all non-undefined array elements are sorted by converting them to strings and comparing strings in UTF-16 code units order. For example, "banana" comes before "cherry". In a numeric sort, 9 comes before 80, but because numbers are converted to strings, "80" comes before "9" in the Unicode order.

If we wanted to sort the numbers numerically, what we can do instead is pass in a compareFunction that sorts it by the value

const a = [1,2,5,7,9];
    const b = [2,5,7,12,100];
    console.log(a.concat(b).sort((a, b) => a - b));

Reduce

For the .reduce() method, it takes in parameters, accumulator and the current value and it returns a single number at the end of the function by applying the rule that was passed in.

Note that it can take in additional values, but these are the main ones we are concerned with

The reducer function takes four arguments: 1. Accumulator 2. Current Value 3. Current Index 4. Source Array

// Reduce
    let array = [1,2,5,7];
    const reducer = (accumulator, currValue) => accumulator + currValue; // Sums = 15
    const getMax = (a,b) => Math.max(a,b); // Returns the largest number in the list
    let a = array.reduce(getMax);
    console.log(a);
    
    //If we wanted the intial value to be something
    let b = array.reduce(reducer, 1); //The sum of the array = 15 + 1

Note that if we don't declare the accumulator it will just take in the first value.

Currying

Ok. So I saw this question which was to add(2)(3) and also add(2,3). When I first saw this, it blew my mind how confusing all of this was.

Then i discovered that this was a technique known as "currying". Okay...so just another crazy nuance using javascript. hahah 👿

Currying is a technique of evaluating a function with multiple arguments, into sequence of function with single/multiple argument. In above problem, we simply turn up add(2,3) into add(2)(3).
//Currying
    function add(x) {
        return function(y) {
            return x + y;
        }
    }
    // Additionally using the ES6 notation
    let add = (x) => (y) => x + y;
    
    // How can we add both of the functions together?
    function add(x, y) {
        if (!(y === undefined)) {
            return x + y;
        } else {
            return function(y) {
                return x + y;
            }
        }
    }
    
    let a = add(2,3);
    let b = add(2)(3);
    console.log(a);
    console.log(b);

Maps vs Objects

They are very different!!

  • For maps, you can store any sort of values in the value of the key value pair
  • Objects: You can ONLY store strings and symbols!

Let's look at a very simple example...

let myMap = new Map();
    myMap.set(1, 'hello');
    myMap.set(2, 'who');
    myMap.set(3, 'am');
    myMap.set(4, 'i');
    
    console.log(myMap);

The following is the result when we console.log() myMap

  • Note that it shows ⇒ to ensure that we know that this is NOT an object
  • We can loop using for..or which can be used for arrays
    • for..in is only used for objects

Map Looping

To loop through keys and values, we do the following

for (let key of myMap.keys()) {
        console.log(key);
    }
    
    
    for (let value of myMap.values()) {
        console.log(value);
    }
    
    for (let [key, value] of myMap) {
        console.log(`The key is ${key} and value is ${value}`);
    }
    
    myMap.forEach((v,k) => console.log(`The key is ${k} and value is ${v}`))
  • myMap.keys()
  • myMap.values()
  • for [key, value] of myMap
  • myMap.forEach(value, key)
    • Note that by using the forEach loop we are usually expected to manipulate the value rather than the key, hence why this is the first parameter.
